# Breeze Viewer Auto-Update Design

**Date:** 2026-04-06
**Status:** Approved

## Overview

Replace the Breeze Viewer's manual update gate (check GitHub Releases → block UI → user downloads installer) with Tauri 2's built-in `tauri-plugin-updater` for fully automatic, silent updates. The app checks for updates on launch, downloads and installs in the background, and the new version takes effect on next restart. No user interaction required.

## Goals

- **Zero-friction updates** — MSP techs should always be on the latest viewer version without manual intervention
- **Signed updates** — All update artifacts are cryptographically signed; the viewer verifies signatures before applying
- **No disruption** — Active remote desktop sessions are never interrupted; updates apply on next launch
- **Zero infrastructure** — Use GitHub Releases as the update endpoint (already in use for distribution)

## Non-Goals

- Update notifications or UI (fully silent)
- Forced restart after update
- Delta/differential updates
- Rollback capability
- Update channels (beta/stable)

## Architecture

### Update Flow

```
App launches
  → setup() spawns async task (3s delay)
    → updater.check() hits GitHub Releases endpoint
      → If no update: done
      → If update found:
        → download_and_install() fetches signed bundle
        → Verifies signature against baked-in public key
        → Stages update for next restart
  → Errors logged, never block usage
  → Next app launch: new version is active
```

### Signing Keys

- **Generate once:** `pnpm tauri signer generate -w ~/.tauri/breeze-viewer.key`
- **Private key:** Stored as GitHub secret `TAURI_SIGNING_PRIVATE_KEY`
- **Private key password (optional):** GitHub secret `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`
- **Public key:** Baked into `tauri.conf.json` under `plugins.updater.pubkey`

### Update Manifest

When `createUpdaterArtifacts: true` is set and `TAURI_SIGNING_PRIVATE_KEY` is provided at build time, Tauri automatically generates:
- Platform-specific update bundles (`.msi.zip` on Windows, `.tar.gz` on macOS/Linux)
- `.sig` signature files for each bundle
- A `latest.json` manifest with version, platform URLs, and signatures

The manifest is uploaded to the GitHub Release alongside the installers. The viewer fetches it from:
```
https://github.com/LanternOps/breeze/releases/latest/download/latest.json
```

### Manifest Format (generated by Tauri)

```json
{
  "version": "0.13.15",
  "notes": "Release notes from GitHub",
  "pub_date": "2026-04-06T12:00:00Z",
  "platforms": {
    "linux-x86_64": {
      "signature": "<base64 sig>",
      "url": "https://github.com/LanternOps/breeze/releases/download/v0.13.15/breeze-viewer-linux.AppImage.tar.gz"
    },
    "windows-x86_64": {
      "signature": "<base64 sig>",
      "url": "https://github.com/LanternOps/breeze/releases/download/v0.13.15/breeze-viewer-windows.msi.zip"
    },
    "darwin-x86_64": {

      "url": "https://github.com/LanternOps/breeze/releases/download/v0.13.15/breeze-viewer-macos.app.tar.gz"

      "signature": "<base64 sig>",

```

## Changes by Layer

### 1. Rust — `apps/viewer/src-tauri/Cargo.toml`

Add dependencies:
```toml
tauri-plugin-updater = "2"
tauri-plugin-process = "2"
```

### 2. Rust — `apps/viewer/src-tauri/src/lib.rs`

Register plugins in the builder chain:
```rust
.plugin(tauri_plugin_updater::init())
.plugin(tauri_plugin_process::init())
```

Add auto-update function:
```rust
async fn auto_update(app: tauri::AppHandle) -> tauri_plugin_updater::Result<()> {
    if let Some(update) = app.updater()?.check().await? {
        eprintln!("Update {} available, downloading...", update.version);
        update.download_and_install(|_, _| {}, || {}).await?;
        eprintln!("Update installed, will apply on next restart");
    }
    Ok(())
}
```

Spawn from `setup()`:
```rust
let update_handle = app.handle().clone();
tauri::async_runtime::spawn(async move {
    tokio::time::sleep(std::time::Duration::from_secs(3)).await;
    if let Err(e) = auto_update(update_handle).await {
        eprintln!("Auto-update check failed: {}", e);
    }
});
```

**Design decisions:**
- **No `app.restart()`** — Never kill active remote desktop sessions. Update applies on next launch.
- **3-second delay** — Let session window creation and WebRTC setup proceed first.
- **Errors swallowed** — Failed update checks must never prevent viewer usage.

### 3. Config — `apps/viewer/src-tauri/tauri.conf.json`

Add to `bundle`:
```json
"createUpdaterArtifacts": true
```

Add to `plugins`:
```json
"updater": {
  "pubkey": "<generated public key>",
  "endpoints": [
    "https://github.com/LanternOps/breeze/releases/latest/download/latest.json"
  ]
}
```

### 4. Frontend — `apps/viewer/src/App.tsx`

Remove all update-related code:
- `UpdateStatus` type
- `updateStatus` and `updateInfo` state
- `checkForUpdate()` useEffect
- `handleOpenDownload` callback
- The entire "Update Required" gate UI block
- `ArrowDownCircle` and `AlertTriangle` icon imports (if unused elsewhere)
- Import of `checkForUpdate` and `UpdateInfo` from `./lib/version`

Simplified flow: detect window label → if session, poll deep link → render `DesktopViewer`.

### 5. Frontend — `apps/viewer/src/lib/version.ts`

Delete this file entirely. All update logic moves to Rust.

### 6. CI — `.github/workflows/release.yml`

In all three viewer build jobs (`build-viewer` Windows/Linux matrix + `build-viewer-macos`):

**Set signing key env vars** (already have `TAURI_SIGNING_PRIVATE_KEY: ''` — change to use the secret):
```yaml
env:
  TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
  TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
```

**Upload `latest.json` to the GitHub Release** alongside existing artifacts. Tauri generates this file in the bundle output directory. The existing artifact rename/upload steps need to also find and upload `latest.json`.

**Important:** All three platform builds produce their own `latest.json` with only their platform's entry. These must be merged into a single `latest.json` before uploading to the release.

**Implementation:** Add a `merge-viewer-update-manifest` job that:
1. Depends on `build-viewer` and `build-viewer-macos`
2. Downloads all per-platform `latest.json` artifacts (named distinctly, e.g. `latest-windows.json`, `latest-linux.json`, `latest-macos.json`)
3. Merges the `platforms` objects into a single `latest.json` (a short `jq` or Node script)
4. Uploads the combined `latest.json` to the GitHub Release via `gh release upload`

Each platform build job uploads its `latest.json` as a distinctly-named artifact (not to the release directly).

### 7. One-Time Setup (Manual)

1. Generate signing keypair: `pnpm tauri signer generate -w ~/.tauri/breeze-viewer.key`
2. Add private key content to GitHub secret `TAURI_SIGNING_PRIVATE_KEY`
3. Optionally add password to `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`
4. Copy public key into `tauri.conf.json`

## Testing

- **Local:** Build with signing key set, verify `latest.json` and `.sig` files are generated
- **CI:** Verify all three platform builds produce signed artifacts
- **End-to-end:** Install an older version, publish a new release, verify the app auto-updates on next launch
- **Failure case:** Disconnect network before update check completes — verify app still works normally
- **Signature mismatch:** Tamper with a bundle — verify updater rejects it

## Security

- Updates are signed with an Ed25519 keypair
- Public key is compiled into the binary — cannot be tampered with at runtime
- Private key never leaves GitHub Secrets
- MITM attacks are prevented by signature verification (even over HTTPS)
- No code execution from untrusted sources — Tauri verifies before applying
